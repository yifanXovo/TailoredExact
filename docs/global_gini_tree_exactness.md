# Global Gini tree exactness argument

## Scope

`--frontier-execution-mode global-gini-tree` is an experimental exact mode. It builds one compact original-problem MILP, loads it into one CPLEX environment and one problem object, calls `CPXmipopt` once, and represents the Gini frontier by recursive child-local branches inside that tree. It does not invoke the fixed-interval oracle, a child process, the Round 18 scheduler, or a time quantum.

The accepted configuration uses one thread, CPLEX presolve on, traditional branch-and-cut search, native best-bound node selection, default heuristics, default probing, and default native cuts. Dynamic search is fail-closed for the reproduced reason recorded in [global_gini_tree_cplex_capability.md](global_gini_tree_cplex_capability.md).

## Root problem and improving range

The root LP is generated by the same compact model writer used by the fixed-interval solver. Its objective is

\[
  \min\; G + \lambda\sum_i w_i e_i.
\]

The only incumbent cutoff at the root is `G + lambda P <= U`, where `U` is independently verified during the same process. It uses no epsilon. Because a verified feasible solution with value `U` remains feasible, this row preserves at least one optimum of the unrestricted problem.

For `V` stations, `0 <= G <= (V-1)/V`. The implementation roots the tree at zero and at an upper bound at least `min(U,(V-1)/V)`. Since `P >= 0`, any solution with `G > U` cannot improve the verified incumbent. The root therefore contains every potentially optimal solution.

## Recursive interval coverage

For a node interval `[L,U]`, the unchanged Round 18 geometry chooses `s` strictly inside the interval and creates exactly

- lower child `[L,s]`; and
- upper child `[s,U]`.

Their union is the parent and their only overlap is the boundary `s`; there is no gap. Induction on branch generations proves coverage for every recursively constructed subtree. Terminal Gini intervals fall back to ordinary CPLEX branching within the same persistent tree.

The callback obtains `L` and `U` from `CPXcallbackgetlocallb` and `CPXcallbackgetlocalub`, so implied local tightening is respected. Every parent/child partition is checked before either child is created. A failed bounds query, coverage check, column mapping, row attachment, or branch API call aborts the callback and prevents certification.

## Global and interval-local strengthening

`IntervalRowFactory` is the single source for the Round 18 static paper-safe profile. Its registry classifies the families as follows.

Global over the root domain:

- inventory conservation;
- movement/reachability domains;
- visit/inventory linking;
- global handling capacity;
- support duration; and
- transfer compatibility.

Interval-local:

- direct Gini cap/floor;
- interval-tight McCormick envelopes;
- objective estimator/cutoff;
- penalty lower-bound closure;
- Gini spread;
- required movement;
- low-Gini centering;
- variable-S centering; and
- the SP product estimator.

The exact G bounds and verified-incumbent-derived inventory bounds are child bound changes. The global families and incumbent row are present in the root model. Each Gini child receives its bounds in `CPXcallbackmakebranch`; all interval-local linear rows are attached at that child's first relaxation by `CPXcallbackaddusercuts` with `purgeable=CPX_USECUT_FORCE` and `local=1`. They are inherited only by descendants of that child and cannot constrain its sibling.

The low-Gini `r_min/r_max` formulation is represented by its exact projection. For every pair, both directions of `r_i-r_j <= B` are present. Their conjunction is exactly `max(r)-min(r) <= B`. The variable-S form is projected identically. This removes presolve-eliminable extrema while preserving the feasible projection and strength of the Round 18 family.

The standalone fixed-interval writer and the global node callback call the same factory with the same instance, options, interval, incumbent, and zero epsilon. Their canonical coefficients, senses, right-hand sides, bound changes, family tags, and signatures are therefore identical. Deterministic tests also check the exact projected-centering equivalence.

## Child estimates and search order

Before custom branching, `CPXcallbackgetrelaxationstatus(..., flags=0)` must return `CPX_STAT_OPTIMAL` or `CPX_STAT_OPTIMAL_INFEAS`. Otherwise the callback records a nonoptimal-relaxation fallback and lets CPLEX branch. For a certified parent relaxation with value `b`, every child feasible region is a subset of the parent, so its relaxation optimum is at least `b`. Supplying `b` as both child estimates is valid. The trace records the parent value and both estimates.

`CPXPARAM_MIP_Strategy_NodeSelect=1` changes only the order in which open nodes are processed. It does not change their union or any constraint and therefore does not affect exactness.

## Native lower bound and certificates

The solver-final status, incumbent, node count, and lower bound all come from the one CPLEX problem. The final lower bound is read with `CPXgetbestobjval`. A time-limited run may report that valid bound and a verified incumbent, but it is not marked optimal.

Optimality is accepted only when all of the following hold:

1. CPLEX returns native integer optimality and a native best bound;
2. the one-environment/one-problem/one-read/one-`CPXmipopt` lifecycle passes;
3. root and recursive interval coverage pass;
4. every required family is registered and migrated;
5. all child bounds and local rows were attached without failure;
6. sibling isolation and column mapping pass;
7. no interval oracle or child process ran; and
8. the independent original-problem verifier accepts the decoded incumbent and objective.

Thus native CPLEX optimality plus the independent verifier yields a certificate for the original problem. Callback checkpoints are diagnostic only and never replace solver-final state.

## Algorithmic exclusions

No branch decision reads an instance name, seed, scale class, known objective, historical result, or Round 18 checkpoint. No fixed seconds select an interval. The split geometry depends only on the current local interval and the existing mechanical frontier settings. Consequently, the exactness argument is independent of the experimental instance matrix and wall-time budget.

## Round 19 mechanical evidence

The retained Round 19 package contains 24 complete global-tree runs: eight
presolve gates, eight 300-second official rows, and eight 900-second official
rows. Across those runs the audits check 2,531 recursive Gini branch events and
4,428 child-local row attachments. Coverage, child-estimate validity, local-row
scope, lifecycle, bound monotonicity, native finalization, certificate source,
and thread fairness all have zero failures.

The child estimates take 1,197 distinct values across the audited tree. The two
siblings of one split deliberately receive the same certified parent
relaxation value; this is valid but provides no sibling discrimination before
their relaxations are processed. It is a performance limitation, not an
exactness limitation.

Three standalone V12_M1 fixed-interval LPs are retained under
`results/gf_global_gini_tree_feasibility_round/reference_models/`. They use
intervals copied from actual global-tree child events, the full frozen static
profile, and the same `round19_v2_projected_centering` factory. Their command
manifests, result JSONs, and SHA-256 fingerprints accompany the deterministic
factory/signature and projected-centering tests.

The authoritative audit rollup is
`results/gf_global_gini_tree_feasibility_round/audit_summary.csv`. Performance
classification is intentionally separate from this exactness evidence and is
reported in the package `final_report.md`.
