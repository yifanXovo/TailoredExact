Problem definition:
There are V stations indexed 1..V and one depot indexed 0. Station i has capacity C_i, initial inventory Y_i^Initial, target inventory Y_i^Target, and weight w_i. There are M identical or near-identical vehicles, each with bicycle capacity Q_k. Every vehicle starts empty from depot 0, visits a sequence of distinct stations, and returns to depot. Across all vehicles, station visits are disjoint: each station is visited by at most one vehicle. At a visited station, a vehicle may pick up or drop off an integer number of bikes. Vehicle load must remain in [0,Q_k]. Station inventory must remain in [0,C_i]. A vehicle may return to the depot with bikes; those bikes are unloaded at the depot and the depot has infinite capacity. The vehicle route duration is travel time plus station pickup/drop-off operation time plus final depot unloading time, and must be <= T for every vehicle. All vehicles start empty.

For final inventory Y_i^Final, define r_i = Y_i^Final / Y_i^Target. The Gini term is

  G = sum_{i=1}^V sum_{j=1}^V |r_i-r_j| / (2 V sum_i r_i)
    = H / (V S),

where H = sum_{i<j}|r_i-r_j| and S = sum_i r_i. The satisfaction penalty is

  P = sum_i w_i |r_i - 1|.

The objective is

  minimize G + lambda * P.

Default parameters for initial tests:
  lambda = 0.15
  T = 3600
  unit_pick_time = 60
  unit_drop_time = 60

If the old project uses a different default convention, detect it and clearly report the convention used.

Phase 2: Tailored exact portfolio.

A. Small-instance full route-load column enumeration mode.
- For V<=8 first, and optionally V<=10 if manageable, enumerate all feasible elementary route-load columns for one vehicle under T and Q.

B. Gini-frontier route-load branch-price-and-cut mode.
- Use route-load columns. The master selects feasible columns; it must not contain arc-flow or MTZ route variables.
- Master constraints: vehicle count, station disjointness, final inventory aggregation, station capacity, r_i, P linearization, and Gini handling.
- For fixed Gini cap gamma, use H <= V*gamma*S to avoid Gini fractional/bilinear expansion.
- Use this as a frontier/bounding engine. Combine it with exact weighted-objective certification via interval/box search over G or a direct exact master objective. Make the certificate explicit in logs.
- Pricing must be exact. Implement at least one of:
  1) C++ label-setting resource-constrained elementary shortest path with load and pickup-budget resources;
  2) compact single-route CPLEX pricing MIP;
  3) both, using label-setting first and MIP pricing as proof fallback.
- A restricted master LP node is certified only when exact pricing proves min reduced cost >= -tolerance.

C. Strengthened compact fallback.
- Keep a strengthened compact CPLEX MILP fallback using globally valid cuts.
- Add operation-time conservation, route-duration lower bounds, symmetry breaking among identical vehicles, and globally valid station/inventory cuts.
- Use it to verify results, obtain incumbents, or close small instances when BPC struggles.

Mandatory valid mathematical components to implement or evaluate:

1. Operation-time conservation.
Since vehicles start empty and unload remaining bikes at the depot,

  operation_time_k = tau_pick * total_pick_k
                   + tau_drop * total_station_drop_k
                   + tau_drop * (total_pick_k - total_station_drop_k)
                   = (tau_pick + tau_drop) * total_pick_k.

For tau_pick=tau_drop=60,

  operation_time_k = 120 * total_pick_k.

This is an equality under the stated assumptions, not a relaxation.

2. Gini cap linearization.

  G <= gamma  iff  H <= V * gamma * S,

where H=sum_{i<j}|r_i-r_j| and S=sum_i r_i. This removes the Gini product/binary expansion in fixed-cap subproblems.

3. Route-load column equivalence.
The set-packing master over all feasible route-load columns is equivalent to the original route/load feasible region when station visits are disjoint and each selected column is internally feasible.

4. Exact pricing certificate.
In Dantzig-Wolfe decomposition, if the exact pricing oracle proves no column has negative reduced cost at a node, the restricted master LP solution is the full master LP solution for that node.

5. 3-subset-row cuts.
For any station triple S,

  sum_c floor(|A_c intersect S|/2) z_c <= 1.

Separate violated cuts dynamically when useful.

6. Co-route / Ryan-Foster pair branching.
For pair (i,j), branch on

  sum_{c containing both i,j} z_c = 0

or

  sum_{c containing both i,j} z_c = 1.

Update master rows and pricing restrictions accordingly.

7. Lorenz/Gini cuts for larger V.
For sorted r_(1)<=...<=r_(V),

  H = sum_l (2l - V - 1) r_(l).

Use dynamic separation if pairwise h_ij becomes too large for V around 50-100.

8. Resource-route bounds.
Use lower bounds on route travel plus operation time to prune infeasible partial routes or columns. For small support sets, exact Held-Karp or DP lower bounds are allowed; for larger V, use MST/1-tree/nearest-return lower bounds.
