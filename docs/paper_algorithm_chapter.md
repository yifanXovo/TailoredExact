# Paper Exact Algorithm Chapter

## Model Recap

For each station final inventory vector `Y`, the objective is
`G(Y) + lambda * P(Y)`. `G` is the Gini-style equity term, `H` denotes the
frontier/interval bookkeeping for admissible Gini ranges, `S` denotes the
station set, and `P` is the service penalty. Feasible route plans must respect
station capacities, vehicle capacity, route duration, operation modes, and the
one-vehicle-per-station service rule used by the exact model.

## Route-Load Column Reformulation

A route-load column represents one vehicle route and its signed pickup/drop
operation vector. The master selects at most one elementary column per vehicle,
enforces station disjointness, and induces the final inventory vector. Route
columns are generated by pricing rather than enumerated globally except in
small guarded diagnostics.

## Gini-Frontier Decomposition

The BPC solves a partition of the Gini range. Each frontier interval has a
lower-bound ledger entry with basis, scope, pricing requirement, and closure
status. The global lower bound is the minimum valid lower bound over active
leaf intervals. The original problem is certified only when all relevant
intervals are closed, skipped, or fathomed and the global gap is zero.

## Master Problem

The restricted master uses elementary route-load columns in production presets.
It enforces vehicle assignment, station disjointness, final-inventory balance,
and branch restrictions. Route-pool incumbent masters are primal heuristics and
use elementary columns only.

## Pricing Subproblem

The production pricing subproblem is exact-label route-load pricing under true
RMP duals and active branch restrictions. Multi-column pricing and projection
dominance may accelerate discovery, but node closure requires exact pricing
semantics: no remaining negative reduced-cost elementary column, unless the
interval has already been fathomed by a valid non-pricing bound.

## Production BPC Components

`paper-bpc-core` includes:

- Projection dominance and filtered multi-column pricing.
- Movement-domain tightening, inventory projection bounds, and penalty-domain
  tightening.
- Vehicle-indexed operation relaxation and vehicle-indexed transfer flow.
- Route-mask operation-budget cuts when based on non-overestimating cycle
  lower bounds.
- Elementary route-pool incumbent master for verified upper bounds only.
- Final-inventory branching and operation-mode branching.

## Exact Portfolio

`paper-exact-portfolio` runs the BPC core and records compact fallback evidence.
The reported certificate module is `bpc`, `compact`, or `none`. A compact row is
certifying only when its own gap is zero and the incumbent verifies.

## Experimental Modules

Hybrid/ng-DSSR pricing, two-track relaxed-RMP columns, and large-instance
diagnostic paths are separated from the production core. They can provide useful
diagnostic progress, but are not paper-core certificate evidence unless their
own closure conditions are satisfied.

## Algorithm 1: paper-exact-portfolio

1. Parse options and expand the selected preset into a `RunConfigSnapshot`.
2. Verify or generate the best incumbent, including archive candidates.
3. Run `paper-bpc-core` and collect the frontier certificate ledger.
4. If requested, run compact fallback as a companion exact module.
5. Select the best verified upper bound and the strongest valid lower bound.
6. Report the certificate module only if one exact module closes with zero gap.

## Algorithm 2: gcap-frontier Route-Load BPC

1. Build the incumbent cutoff and Gini frontier intervals.
2. For each interval, compute cheap valid bounds and inventory/route/Gini bounds.
3. If an interval is not fathomed, run elementary branch-price with exact
   pricing and branch restrictions.
4. Update the active leaf ledger, adaptive splits, focused intensification, and
   progress checkpoints.
5. Aggregate the global lower bound from valid interval bounds.

## Algorithm 3: Route-Load Pricing

1. Read true RMP duals and active branch restrictions.
2. Generate elementary route-load labels respecting time, capacity, operation
   mode, support duration, and branch constraints.
3. Return negative reduced-cost columns after feasibility verification.
4. For closure, exhaust or validly dominate all labels and prove no negative
   reduced-cost elementary column remains.

## Algorithm 4: Certificate Ledger

1. For each interval, record certificate basis, lower-bound source list,
   pricing closure requirement, pricing closure availability, and bound validity.
2. Reject certification when any relevant interval is unresolved or invalid.
3. Reject certification when the option audit fails, the verifier fails, or the
   final gap is positive.
4. Report diagnostic scope for focus-only, large-instance, relaxed-RMP
   incomplete, seeded, imported-incumbent, and capped runs.
