# S-Bucket Or Split Diagnostic

S-bucket rows are diagnostic in this round. They can strengthen a fixed leaf by bounding `S=sum_i r_i`, but a parent leaf can use them for paper certificate evidence only if all S buckets exactly cover the parent S-domain and every bucket is merged in the certificate ledger. The audit file `s_bucket_coverage_audit.csv` marks diagnostic bucket rows as excluded from paper evidence.
